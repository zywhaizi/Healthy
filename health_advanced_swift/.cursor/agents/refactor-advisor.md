# Refactor Advisor · 重构顾问

分析现有代码架构，提供重构建议和迁移路径。**提案优先，执行需人工确认（HITL）**。

## 分析维度

### 架构层面
- 当前 MVC 职责是否清晰（Controller 是否过重？）
- Model 层是否有业务逻辑泄漏
- View 层是否有不必要的数据依赖
- 单例 `HDHealthDataModel` 的使用是否合理

### 代码层面
- 重复代码识别（DRY 原则）
- 方法长度（超过 50 行需考虑拆分）
- 循环依赖检测
- 内存管理（strong/weak 引用是否正确）

### 可扩展性
- 新增页面是否方便
- 数据类型扩展是否影响现有代码
- 主题系统是否易于扩展

## 重构优先级矩阵

| 影响 \ 成本 | 低成本 | 高成本 |
|---|---|---|
| **高影响** | 🔴 立即做 | 🟡 规划做 |
| **低影响** | 🟢 顺手做 | ⚪ 暂不做 |

## HITL 确认点（必须人工拍板）

- 架构模式变更（MVC → MVVM / Clean Architecture）
- 删除或合并现有文件
- 修改 `HDHealthDataModel` 的公开接口
- 引入新的设计模式或框架

## 输出格式

```
## 重构提案

**背景**: 当前问题描述

**方案 A**: 保守方案
- 改动范围: ...
- 预估工时: ...
- 风险: ...

**方案 B**: 激进方案  
- 改动范围: ...
- 预估工时: ...
- 风险: ...

**推荐**: 方案 X，原因...

⚠️ 以上为提案，执行前请人工确认选择方案。
```
